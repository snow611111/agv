# 基于大模型的巡检报告生成模块

## 1. 模块概述

本模块实现轨道交通巡检报告的自动生成功能。手持终端在完成任务巡检后，将任务信息和故障列表提交至 Dify 工作流，由本地大模型（qwen3:8b）生成结构化的正式巡检报告。

### 核心能力

- 接收任务 JSON + 故障列表 JSON，输出正式巡检报告
- 故障统计（总数、已确认、未确认）
- 字段缺失自动标记"数据未提供"
- 未确认故障自动提示人工复核
- 处理建议仅基于已有数据生成，不虚构专业措施

### 适用场景

鸿蒙手持终端 → 获取 AGV 任务和故障数据 → 调用本模块 → 展示 AI 巡检报告

### 免责声明

> 本报告由 AI 生成，仅供辅助参考，最终以人工复核为准。

## 2. 技术架构

```
鸿蒙手持终端
      │
      │ POST /v1/workflows/run
      ▼
  Dify 平台（Docker 部署）
      │
      │ 工作流：开始 → LLM(qwen3:8b) → 结束
      │ 输入：task_info, flaw_list, upload_info
      │ 输出：report_content
      ▼
  Ollama（本地大模型引擎）
      │
      ▼
  qwen3:8b（5.2 GB，CPU 推理）
```

## 4. 部署步骤

### 4.1 安装 Docker Desktop（Windows）

### 4.2 克隆 Dify

```bash
git clone https://github.com/langgenius/dify.git
cd dify/docker
cp .env.example .env
```

### 4.3 配置 Dify 的 .env

在 `.env` 文件末尾添加：

```
CUSTOM_MODEL_ENABLED=true
OLLAMA_API_BASE_URL=host.docker.internal:11434
```

### 4.4 启动 Dify

```bash
docker compose up -d
```

### 4.5 安装 Ollama 并拉取模型

```bash
ollama serve
ollama pull qwen3:8b
```

### 4.6 配置 Dify

1. 访问 `http://localhost/install` 完成初始化
2. 设置 → 模型供应商 → Ollama → 添加 `qwen3:8b`
3. 创建应用 → 选择"工作流" → 导入 sync_payload.json
4. 发布 → 访问 API → 获取 API Key

## 6. 提示词设计要点

提示词经过多轮迭代优化，核心约束：

1. **输出控制**：禁止 `<think>` 标签，只输出最终报告正文
2. **字段映射**：execTime→开始时间，endTime→结束时间，taskTrip→里程（米）
3. **真实性约束**：只引用输入中存在的字段，禁止推测和虚构
4. **处理建议**：限定四类（人工复核、补充记录、持续观察、提交专业人员）
5. **数据完整性**：以结论形式撰写，不罗列 JSON 字段名
6. **结论审慎**：不写无法验证的断言

## 7. 工作流结构

```
[开始] ────→ [LLM: qwen3:8b] ────→ [结束]
  │                │                    │
  ├─ task_info     ├─ 温度 0.2         └─ report_content
  ├─ flaw_list     ├─ max_tokens 4096
  └─ upload_info   └─ 提示词（system prompt）
```

---

## 8. 字段映射关系

| JSON 字段 | 报告中名称 | 说明 |
|-----------|-----------|------|
| taskCode | 任务编号 | |
| taskName | 任务名称 | ≠ 巡检类型 |
| executor | 执行人 | |
| taskStatus | 任务状态 | 待巡视/巡视中/待上传/已完成/执行中 |
| execTime | 巡检开始时间 | 可选 |
| endTime | 巡检结束时间 | 可选 |
| taskTrip | 巡检里程 | 单位：米 |
| flawName | 故障名称 | |
| flawType | 故障类型 | 结构缺陷/渗漏问题/设备故障等 |
| flawDesc | 故障描述 | 可选 |
| flawDistance | 故障位置 | 单位：米 |
| confirmed | 确认状态 | true/false |
